PERTANYAAN UMUM SIDANG & JAWABAN
==============================================

1. PERTANYAAN TENTANG LATAR BELAKANG
====================================

Q1: "Mengapa memilih perpustakaan sebagai objek penelitian?"
A1: "Perpustakaan SMK YMIK masih menggunakan sistem manual dengan pencatatan buku besar. Ini menyebabkan kesulitan dalam pencarian data, rawan kehilangan catatan, dan proses peminjaman-pengembalian yang lama. Sistem digital akan meningkatkan efisiensi dan akurasi."

Q2: "Apa masalah utama sistem manual yang ada?"
A2: "Masalah utama: 
- Pencarian data lambat dan sulit
- Risiko kehilangan catatan fisik
- Kesulitan monitoring stok dan peminjaman
- Laporan manual membutuhkan waktu lama
- Tidak ada backup otomatis
- Proses pelayanan siswa kurang efisien"

Q3: "Mengapa tidak menggunakan sistem perpustakaan yang sudah ada?"
A3: "Sistem existing umumnya mahal dan kompleks untuk SMK. Sistem custom ini disesuaikan dengan kebutuhan spesifik SMK YMIK, lebih cost-effective, dan dapat dikustomisasi sesuai perkembangan kebutuhan sekolah."

2. PERTANYAAN METODOLOGI
=======================

Q4: "Mengapa memilih metode waterfall/agile?"
A4: "Memilih waterfall karena:
- Requirement sudah jelas dari awal
- Timeline proyek fixed
- Team development kecil
- Budget terbatas
- Stakeholder menginginkan deliverable yang jelas di setiap fase"

Q5: "Bagaimana proses requirement gathering?"
A5: "Proses requirement gathering:
- Interview dengan kepala perpustakaan dan petugas
- Observasi proses manual existing
- Analisis dokumen dan form yang digunakan
- Survey kepuasan user terhadap sistem manual
- Benchmarking dengan sistem perpustakaan lain"

Q6: "Apa tools yang digunakan untuk analisis dan design?"
A6: "Tools yang digunakan:
- Microsoft Visio untuk flowchart dan DFD
- MySQL Workbench untuk database design
- Draw.io untuk use case diagram
- Microsoft Project untuk project timeline
- VS Code untuk development"

3. PERTANYAAN TEKNIS
===================

Q7: "Mengapa memilih PHP dan MySQL?"
A7: "Alasan memilih PHP-MySQL:
- PHP mudah dipelajari dan banyak dokumentasi
- MySQL stable dan reliable untuk aplikasi medium
- Hosting murah dan mudah ditemukan
- Banyak community support
- Cocok untuk rapid development
- Integration yang baik antara PHP-MySQL"

Q8: "Bagaimana menangani security sistem?"
A8: "Security measures yang diimplementasi:
- Password hashing untuk authentication
- Prepared statements untuk prevent SQL injection
- Input validation dan sanitization
- Session management dengan timeout
- Role-based access control
- HTTPS untuk data transmission
- Regular backup untuk data protection"

Q9: "Bagaimana sistem menangani concurrent users?"
A9: "Handling concurrent users:
- PHP naturally support multi-user
- MySQL innodb engine support concurrent connections
- Database transaction untuk data consistency
- Session isolation untuk setiap user
- Optimistic locking untuk prevent race condition"

Q10: "Apa yang dilakukan jika database crash?"
A10: "Disaster recovery plan:
- Regular automated backup (daily)
- Multiple backup locations (local + cloud)
- Database replication jika memungkinkan
- Recovery procedure dengan RTO < 1 hour
- Data integrity verification after restore
- Rollback plan jika restore gagal"

4. PERTANYAAN FUNGSIONAL
=======================

Q11: "Fitur apa saja yang ada di sistem?"
A11: "Fitur utama sistem:
- Master data (buku, siswa, petugas)
- Transaksi (peminjaman, pengembalian)  
- Manajemen denda otomatis
- Laporan dan statistik
- User management dengan role
- Dashboard dengan overview
- Search dan filter data
- Print receipt dan laporan"

Q12: "Bagaimana sistem menghitung denda?"
A12: "Perhitungan denda:
- Denda keterlambatan: Rp 1000 per hari
- Denda kerusakan: Rp 10.000 flat
- Denda kehilangan: 100% harga buku
- Sistem otomatis hitung based pada tanggal kembali
- Status denda (lunas/belum lunas)
- History pembayaran denda"

Q13: "Apa business rules yang diterapkan?"
A13: "Business rules:
- Maksimal 3 buku per siswa
- Durasi peminjaman 7 hari
- Siswa harus status aktif untuk meminjam
- Tidak bisa pinjam jika ada denda tertunggak
- Stok buku harus tersedia
- Auto logout setelah 30 menit idle"

TIPS MENJAWAB PERTANYAAN:
========================

1. Dengarkan pertanyaan sampai selesai
2. Pause sebentar untuk think
3. Jawab dengan struktur: Point utama → Detail → Example
4. Gunakan data dan fakta jika ada
5. Jika tidak tahu, jujur dan offer research
6. Stay confident tapi humble
7. Thank setiap pertanyaan
8. Connect ke business value
9. Show technical competency
10. Demonstrate problem-solving skill

PERSIAPAN:
=========
- Practice dengan teman/keluarga
- Record diri sendiri menjawab
- Prepare data dan statistik
- Know sistem inside-out
- Understand business context
- Stay updated dengan technology trend
- Prepare demo yang smooth
- Backup plan jika demo gagal

Remember: Tujuan sidang adalah evaluasi kemampuan Anda dalam problem solving, technical skill, dan communication. Show passion dan ownership terhadap project!
